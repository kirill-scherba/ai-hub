A unique MCP server that lets AI assistants create, execute, export, import, and share new MCP tools **dynamically**, using Perl's built-in [Safe](https://metacpan.org/pod/Safe) sandbox (since 1994, no external dependencies).

## Why?

All existing "tool marketplaces" are static — tools are written by humans, submitted, and installed. This hub turns that upside-down:

1. **AI writes tools** — an AI assistant can generate a Perl function at runtime
2. **Safe sandbox** — code runs in `Safe->new()` with whitelisted modules only
3. **Instant registration** — the new tool appears in `tools/list` immediately
4. **Export/Import** — tools can be saved as JSON and shared with other AIs
5. **Hub Sharing** — publish, search, and pull tools from a central server

## Built-in Tools

| Tool | Description |
|---|---|
| `tool_generate` | Create a new MCP tool with name, schema, and Perl code |
| `tool_list` | List all tools (built-in + generated) |
| `tool_export` | Export a generated tool as JSON |
| `tool_import` | Import a tool from JSON |
| `tool_remove` | Remove a generated tool |
| `hub_publish` | Publish a local tool to the Hub Server |
| `hub_search` | Semantic search for tools on the Hub |
| `hub_pull` | Download a tool from the Hub and install locally |
| `hub_list` | List tools available on the Hub |

## Quick Start

### Using Perl directly

```bash
# Run the hub (stdio mode for MCP)
perl generative-mcp-hub.pl
```

### Using Docker

```bash
# Build and run just the Perl hub
docker build -t ai-hub .
docker run -i --rm ai-hub

# Or run both hub-server + Perl hub together
docker compose up --build
```

### MCP Client Configuration

AI Hub communicates via **stdio** — the standard MCP transport. Configure your MCP client to launch the hub as a subprocess.

**Direct stdio connection** (Perl process spawned by the client):

```json
{
  "mcpServers": {
    "ai-hub": {
      "command": "perl",
      "args": ["/path/to/ai-hub/generative-mcp-hub.pl"]
    }
  }
}
```

This is the cleanest mode — no wrappers, no containers, just the Perl process talking stdio with your AI assistant.

**Containerized stdio** (via Docker):

```json
{
  "mcpServers": {
    "ai-hub": {
      "command": "docker",
      "args": ["run", "-i", "--rm", "ai-hub"]
    }
  }
}
```

## Example: Generate a Tool

Once connected, tell your AI assistant:

> "Use tool_generate to create a tool called 'weather_check' that takes a city name and returns a simulated weather report."

The AI will craft a Perl function, the hub will compile it in the Safe sandbox, and the new `weather_check` tool becomes instantly available.

### Real Generated Tool Example

Here is an actual tool definition that was generated at runtime by an AI assistant — `convert_units`:

```json
{
  "name": "convert_units",
  "description": "Convert between measurement units: length, weight, temperature, volume",
  "inputSchema": {
    "type": "object",
    "required": ["value", "from", "to"],
    "properties": {
      "value":   { "type": "number", "description": "The value to convert" },
      "from":    { "type": "string", "description": "Source unit code" },
      "to":      { "type": "string", "description": "Target unit code" }
    }
  }
}
```

The Perl code behind it (simplified):

```perl
my ($v, $f, $t) = @{$args}{qw(value from to)};
# ... unit conversion logic (length/weight/temperature/volume)
return { result => $v * $m{$f} / $m{$t} + 0.0, from => $f, to => $t, ... };
```

## Hub Server

The Hub Server is a Go HTTP REST API server that provides a marketplace for MCP tools with semantic search.

```bash
# Start the hub server (requires Go)
cd hub-server && go run . --port 8484
```

### Hub API Endpoints

| Method | Endpoint | Description |
|---|---|---|
| `GET` | `/tools` | List all tools (optional `?prefix=`) |
| `GET` | `/tools/{name}` | Get a single tool by name |
| `POST` | `/tools` | Create/publish a tool |
| `DELETE` | `/tools/{name}` | Delete a tool |
| `GET` | `/search?q=...&limit=N` | Semantic search (requires Ollama) |

### Hub Workflow

```bash
# 1. Start the hub server
cd hub-server && go run . --port 8484 &

# 2. Start the Perl MCP server with Hub URL
perl generative-mcp-hub.pl --hub-url http://localhost:8484

# 3. From your AI assistant, use hub_publish to share tools,
#    hub_search to find tools semantically, hub_pull to install them.
```

## Sandbox & Security

Code generated by `tool_generate` runs in a Perl `Safe` sandbox with strict restrictions:

- **Whitelisted modules only**: `MIME::Base64`, `Digest::MD5`, `URI::Escape`, `Scalar::Util`, `Cwd`, `Time::Piece`, `JSON::PP`, `Encode`
- **HTTP helpers shared**: `_safe_http_get`, `_safe_http_get_json`, `_uri_escape_utf8`, `_safe_curl_get_http_code`, `_safe_curl_get_code_and_time`, `_safe_curl_verbose`
- **Permitted ops**: `time`, `rand`, `srand`
- **Blocked**: `system`, `exec`, `open` (file I/O), `require` (unlisted modules), network sockets, shell access

> See [docs/SECURITY.md](docs/SECURITY.md) for a detailed sandbox analysis.

## Why Perl?

Only Perl has `Safe->new()` built into its core since 1994. No Python, JavaScript, or Lua offers a comparable built-in sandbox without external dependencies.

- **Python**: `exec()` with custom globals is trivial to escape; third-party libs required
- **JavaScript**: `vm2` is deprecated; `vm` module is insecure
- **Lua**: sandbox requires manual metatable restrictions
- **Perl**: `Safe->new()` is a first-class core module, actively maintained, with compartment-based security
