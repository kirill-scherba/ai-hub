# Security Model — Generative MCP Hub

## Overview

The core security mechanism is Perl's built-in `Safe` module — a compartment-based execution sandbox that has been part of Perl's core since Perl 5 (1994). All code generated by `tool_generate` is compiled and executed inside a `Safe` compartment.

## Sandbox Architecture

```
┌─────────────────────────────────────┐
│           MCP Hub (Perl)            │
│  ┌───────────────────────────────┐  │
│  │        Safe Compartment        │  │
│  │  ┌─────────────────────────┐  │  │
│  │  │  Generated Tool Code     │  │  │
│  │  │  (user-provided Perl)    │  │  │
│  │  └─────────────────────────┘  │  │
│  │  ┌─────────────────────────┐  │  │
│  │  │  Shared Functions:      │  │  │
│  │  │  _safe_http_get         │  │  │
│  │  │  _safe_http_get_json    │  │  │
│  │  │  _uri_escape_utf8       │  │  │
│  │  │  _safe_curl_*           │  │  │
│  │  └─────────────────────────┘  │  │
│  └───────────────────────────────┘  │
└─────────────────────────────────────┘
```

## Whitelisted Modules

Only the following Perl modules can be loaded via `use` inside generated code:

| Module | Purpose | Risk |
|---|---|---|
| `MIME::Base64` | Base64 encoding/decoding | ✅ Minimal; no I/O |
| `Digest::MD5` | MD5 hashing | ✅ Minimal; no I/O |
| `URI::Escape` | URL encoding | ✅ Minimal; no I/O |
| `Scalar::Util` | Scalar utilities (`blessed`, `refaddr`, etc.) | ✅ Minimal; introspection only |
| `Cwd` | Current working directory (read-only) | ✅ Read-only path info |
| `Time::Piece` | Date/time formatting | ✅ No I/O or system access |
| `JSON::PP` | JSON parsing/generation | ✅ Pure-Perl, no I/O |
| `Encode` | Character encoding | ✅ Pure-Perl, encoding only |

Any attempt to `use` a module not in this whitelist will be rejected with:
```
Module 'Some::Module' is not allowed in Safe sandbox.
```

## Shared Functions (Exposed to Sandbox)

These functions are shared from the main namespace into the Safe compartment:

| Function | Description | Security Notes |
|---|---|---|
| `_safe_http_get` | Fetches URL content via curl | ⚠️ Network access (required for tools) |
| `_safe_http_get_json` | Fetches and decodes JSON from URL | ⚠️ Network access (required for tools) |
| `_uri_escape_utf8` | URL-encodes a string | ✅ No I/O |
| `_safe_curl_get_http_code` | Returns HTTP status code | ⚠️ Network access |
| `_safe_curl_get_code_and_time` | HTTP code + response time | ⚠️ Network access |
| `_safe_curl_verbose` | Verbose curl output (for SSL info) | ⚠️ Network access |

## Permitted Opcodes

By default, `Safe` denies all opcodes. The sandbox explicitly permits:

- `time` — current time (for timestamps)
- `rand` / `srand` — random numbers

All other opcodes — including `system`, `exec`, `backtick`, `fork`, `dbmopen`, `open`, `sysopen`, `require`, `do`, `eval` — are **denied by default** unless inherited from shared code.

## What Is Blocked

| Attack Vector | Blocked? | Mechanism |
|---|---|---|
| `system()`, `exec()` | ✅ Blocked | Default Safe restrictions |
| Backtick (`` ` ``) commands | ✅ Blocked | Default Safe restrictions |
| File I/O (`open`, `sysopen`) | ✅ Blocked | Default Safe restrictions |
| Loading arbitrary modules | ✅ Blocked | Whitelist check before Safe compilation |
| Accessing main package symbols | ✅ Blocked | Safe compartment isolation |
| Infinite CPU loops | ⚠️ No protection | Use OS-level ulimit/timeout |
| Memory exhaustion | ⚠️ No protection | Use OS-level ulimit |

## Best Practices for Generated Code

1. **Avoid hardcoded secrets** — any code submitted to `tool_generate` is stored in `tools.json` as plaintext. Treat it as public.

2. **Limit network access** — generated tools use `_safe_http_get` which is rate-limited by the external API, not by the sandbox.

3. **Validate inputs** — the `inputSchema` parameter of `tool_generate` serves as documentation; actual validation is the code's responsibility.

4. **Use only whitelisted modules** — if your tool needs additional modules, consider adding them to the whitelist in `compile_in_safe()`.

## Risk Assessment

| Risk | Severity | Mitigation |
|---|---|---|
| Malicious code execution | Low | Safe sandbox prevents system commands, file I/O, and arbitrary module loading |
| Network abuse (DoS external API) | Low-Medium | Generated tools can only make HTTP GET requests; manual review recommended before publishing to Hub |
| Information disclosure | Low | Sandbox cannot read files; tool code in `tools.json` is readable by anyone with filesystem access |
| Escape from Safe | Very Low | Safe is a well-audited core Perl module; no known escapes in recent versions |

## Limitations

- **CPU/memory DoS**: `Safe` does not limit CPU time or memory. For production deployments, wrap the MCP hub with OS-level resource limits (e.g., `ulimit`, systemd `CPUQuota`, Docker memory limits).
- **Network access**: Generated tools have access to HTTP/HTTPS via shared curl functions. They cannot open raw sockets but can make arbitrary HTTP requests.
- **No deep code analysis**: The sandbox does not analyze what the code *does* — only what Perl operations it *uses*. A tool could legitimately fetch data from any URL.
